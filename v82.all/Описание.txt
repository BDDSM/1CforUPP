Инструкция по работе справки:
    Слева раздел справки, с права краткое описание раздела.
    !- Данный символ означает, что данный раздел не реализован.
    ?- Данный символ означает, что справка по данному разделу отсутствует.
    Для обращения к разделу нужно выполнить хранимую процедуру, например вот так "v82Help 'Установка'" или "exec v82Help 'v82Help'".

0) Служебные процедуры и функции:
    Название                    - Выбор названия данной сборки.
    Проблемы                    - Список проблем, которые сборка может помочь решить.
    Задачи                      - Список задач, которые сборка решает.
    Преимущества                - Плюсы сборки относительно внешней компоненты.
    Установка                   - Инструкция по установке сборки.
    Удаление                    - Инструкция по удалению сборки.
    ?Обновление                 - Инструкция по обновлению сборки.
    V82Install                  - Уcтановка процедур и функций.
    !V82Clear                   - Очистка кэша сборки.
    V82Help                     - Справка по работе с процедурами и функциями.    
    V82View                     - Разворачивание представлений(View).
    !V82Index                   - Работа с sql-индексами.
    V82Grant                    - Выдача прав на процедуры и функции учетной записи базы данных(sysusers), которой необходим доступ к возможностям сборки.
    !V82Script                  - Коллекция полезных скриптов.
    sp_Config.V82.Sql           - Процедура, хранящая строки подключений к sql-базам. Процедура должна возвращать sql-сборке идентификаторы баз и строки подключения к ним.
    !sp_LogException.V82.Sql    - Если процедура существует в нее будут передаваться описание исключений, возникших при использовании доступа к внешним sql-базам.     
    sp_Config.V82.Com           - Процедура, хранящая строки подключений к 1c-базам.
    !sp_LogException.V82.Com    - Если процедура существует в нее будут передаваться описание исключений, возникших при использовании доступа к внешним 1c-базам. 
    !sp_Config.V82.Xls          - Процедура, хранящая каталоги для работы с Excel-файлами.
    !sp_Config.V82.Xml          - Процедура, хранящая карты конвертации для xml-данных.
    sp_Config.V82.Tcp           - Процедура, хранящая настройки tpc-серверов.
    sp_Config.V82.Web           - Процедура, хранящая настройки web-серверов.
    V82ЗакрытиеПараметра        - Функция для шифровки параметров.
    !V82PublicKey               - Процедура возвращает открытые ключи.
    !V82EnCrypt                 - Процедура может использоваться для шифрования.   
    !V82DeCrypt                 - Процедура может использоваться для расшифровки.
1) Процедуры и функции для выполнения sql-запросов:
    V82ListSQL          - Список баз и строк соединений(пароли удаляются).
    V82Pack             - Выполнить группу запросов с описанием схемы результата. Так же выполнение группы запросов для набора параметров. Для лучшего понимания лучше прочесть дополнительные разделы справки 'V82Pack' и 'Типизация'.
    V82PrepareQuery     - Выполняет предварительную обработку текста запроса. Переводит виртуальные(логические) имена в физические. 
2) Процедуры и функции для com-обращений к 1С:Предприятие:      
    Требования:
        - На сервере с "SQL Server" должен быть установлен com-коннектор 1С.
    Нагрузочные тесты:
        - Оболочка должна держать 1000 подключений, со скоростью вызова функций 300-400 в секунду.  
    Выполненение функций:
    V82Eval             - Вычислить арифметическое или логическое выражение на языке 1С(Вычислить). Функции 1с-платформы и 1с-конфигурации так же доступны.  
    V82Execute          - Выполнить код на языке 1С.
    V82Query            - Выполнить запрос на языке запросов 1С. Возвращает в sql-сервер результат запроса в табличном виде. Результат может быть использован в хранимых процедурах.
    V82Call             - Выполнить функцию внешнего соединения. Функция 1С должна соответстовать формату "Функция(Параметр)".
    V82CallModule       - Выполнить функцию общего модуля. Функция 1С должна соответстовать формату "Функция(Параметр)".
    V82CallProcessor    - Выполнить функцию обработки. Функция 1С должна соответстовать формату "Функция(Параметр)".
    V82CallExtProcessor - Выполнить функцию внешней обработки. Функция 1С должна соответстовать формату "Функция(Параметр)".
    Работа с метаданными:
    V82ActiveUsers      - Вернуть список активных пользователей.
    V82Users            - Вернуть список пользователей.
    V82Enums            - Вернуть список перечислений.
    V82Catalogs         - Вернуть список справочников.
    V82Documents        - Вернуть список документов.
    V82InfRegisters     - Вернуть список регистров сведений.
    V82AccRegisters     - Вернуть список регистров накопления.
    ?V82Reports          - Вернуть список отчетов.
    V82Processors       - Вернуть список обработок.
    Работа с пулом:
    !V82ConfigSQL        - Вернуть список зарегистрированных хранимой процедурой [sp_Config.V82.Sql] sql-баз. Строки подключения расшифровываются, пароли вырезаются.
    !V82ConfigCOM        - Вернуть список зарегистрированных хранимой процедурой [sp_Config.V82.Com] com-баз. Строки подключения расшифровываются, пароли вырезаются.
    V82PoolList         - Вернуть список баз и статистику пула для каждой. (Размер Пула,Время самого старого соединения в пуле, Время самого нового соединения в пуле, Время первого вызова, Время последнего вызова, Количество вызовов, Среднее время вызова)
    !V82PoolClose        - Закрытие пула, отключение всех соединений.  
    !V82PoolShrink       - Сжатие пула, отключение соединений, последний вызов которых был до ТекущаяДата() минус отступ(Параметр).

4) Процедуры и функции для работы с закрытыми(паролями) модулями:
    V82ЗакрытыйМодульКонстанты          - Возвращает список используемых констант и переменных в закрытом модуле.
    V82ЗакрытыйМодульПеременные         - Возвращает список переменных(глобальных) в закрытом модуле. 
    V82ЗакрытыйМодульФункции            - Возвращает таблицу свойств функций( Имя, ВозвращаетЗначение, Экспорт, Встроенная, КоличествоПараметров, НомерПервойСтроки) в закрытом модуле.
    V82ЗакрытыйМодульКоманды            - Возвращает список команд функции из закрытого модуля с использованием алгоритма восстановления кода.
    V82ЗакрытыйМодульВыполнитьФункцию   - Выполняет функции из закрытого модуля без подключения к 1С.    
5) Процедуры и функции для чтения внутренних значений 1С:
    V82Parse        - Распаковывает внутреннее значение в виде таблицы.
    !V82ParseToXML  - Распаковывает внутреннее значение в виде xml.
    !               - Распаковывает внутреннее значение в виде табличного представления(Все поля представленны бинарными потоками и количество полей фиксировано).  
6) !Процедуры и функции для сериализации и загрузки данных в базу.
    1) Сериализация справочников и их загрузка. Варианты сериализации.
    2) Подготовка объектов на строне .Net, передача коллекцией в 1С.
    3) Прямая запись в базу эфективнее. Можно открыть транзакцию, вызвать 1С, и, если все верно, зафиксировать транзакцию.
